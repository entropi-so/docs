---
title: Environment variables
description: "Every variable Entropi reads at startup. Required values, defaults, and what they control."
---

Entropi reads its configuration from a `.env` file (or the process environment) at startup. Pydantic validates required fields — the server will refuse to boot if a required variable is missing or malformed.

## Required

| Name                          | Type    | Default | Description                                                                                          |
| ----------------------------- | ------- | ------- | ---------------------------------------------------------------------------------------------------- |
| `DATABASE_URL`                | string  | —       | PostgreSQL connection string. Format: `postgresql://user:pass@host:port/db`.                         |
| `SECRET_KEY`                  | string  | —       | JWT signing secret. Minimum 32 characters. Generate with `python -c "import secrets; print(secrets.token_urlsafe(32))"`. |
| `GEMINI_VISION_API_KEY`       | string  | —       | Gemini API key used by the capture pipeline (visual understanding + crop).                           |
| `GROQ_ADVISOR_API_KEY`        | string  | —       | Groq API key used by the advisor service.                                                            |
| `GROQ_URL_IMPORT_API_KEY`     | string  | —       | Groq API key used by the URL import service.                                                         |
| `GOOGLE_CLIENT_ID`            | string  | —       | OAuth client ID for Google sign-in.                                                                  |
| `GOOGLE_CLIENT_SECRET`        | string  | —       | OAuth client secret for Google sign-in.                                                              |
| `GOOGLE_REDIRECT_URI`         | string  | —       | OAuth redirect URI (must match the one registered with Google).                                      |

## Optional

| Name                            | Type    | Default | Description                                                                                     |
| ------------------------------- | ------- | ------- | ----------------------------------------------------------------------------------------------- |
| `VOYAGE_API_KEY`                | string  | —       | Voyage AI key for multimodal embeddings (semantic search). Without it, capture still works but products are left unindexed and `GET /products/search` returns `503`. |
| `ALGORITHM`                     | string  | `HS256` | JWT signing algorithm.                                                                           |
| `ACCESS_TOKEN_EXPIRE_MINUTES`   | integer | `10080` | Token lifetime in minutes (default: 7 days).                                                     |
| `LOG_LEVEL`                     | string  | `INFO`  | One of `DEBUG`, `INFO`, `WARNING`, `ERROR`.                                                      |
| `HOST`                          | string  | `0.0.0.0` | Bind address.                                                                                  |
| `PORT`                          | integer | `8000`  | Bind port.                                                                                       |
| `CORS_ORIGINS`                  | string  | —       | Comma-separated list of allowed origins.                                                         |

<Note>
  `VOYAGE_API_KEY` is intentionally optional and is **not** flagged at startup,
  so the server boots (and captures) without it. Semantic search is the only
  feature it gates.
</Note>

## Example `.env`

```bash
# Database
DATABASE_URL=postgresql://postgres:password@localhost:5432/palme

# AI providers
GEMINI_VISION_API_KEY=...
GROQ_ADVISOR_API_KEY=...
GROQ_URL_IMPORT_API_KEY=...
VOYAGE_API_KEY=...            # optional — enables semantic search

# OAuth
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...
GOOGLE_REDIRECT_URI=http://localhost:3000/callback

# Auth
SECRET_KEY=replace-me-with-a-32-char-random-string
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=10080

# Logging
LOG_LEVEL=INFO
```

## Verifying configuration

```bash
python -c "
import os
from dotenv import load_dotenv
load_dotenv()

required = [
    'DATABASE_URL',
    'SECRET_KEY',
    'GEMINI_VISION_API_KEY',
    'GROQ_ADVISOR_API_KEY',
    'GROQ_URL_IMPORT_API_KEY',
    'GOOGLE_CLIENT_ID',
    'GOOGLE_CLIENT_SECRET',
    'GOOGLE_REDIRECT_URI',
]
missing = [v for v in required if not os.getenv(v)]
print(f'Missing: {missing}' if missing else 'All required variables set')
print('Semantic search:', 'enabled' if os.getenv('VOYAGE_API_KEY') else 'disabled (no VOYAGE_API_KEY)')
"
```

## Security

<Warning>
  Never commit `.env` to version control. Add `.env`, `.env.local`, and `.env.*.local` to `.gitignore`.
</Warning>

Rotate the Gemini, Groq, Voyage, and OAuth credentials on a defined cadence. Keep `SECRET_KEY` distinct per environment — never share dev and prod values.

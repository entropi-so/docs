## Endpoint

```http
GET /products/search
```

## Overview

Free-text semantic search over the caller's **own** products. The query is
embedded (query-side) and compared against each product's multimodal
embedding, so it matches on what the photo *shows* — color, object, setting —
as much as on the stored text. A search for *"cozy green jacket"* finds a
green jacket even if none of those words appear in its record.

Results are always scoped to `Product.user_id` (derived from the JWT, never
the client) and ranked by relevance descending. Products not yet indexed (no
embedding) are simply not returned.

## Request

### Authentication

```http
Authorization: Bearer <access_token>
```

### Query parameters

<ParamField query="q" type="string" required>
  The free-text query. Text only — never an image, never reverse image
  search. Must be non-empty.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Page size, 1–100.
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Pagination offset.
</ParamField>

### cURL

<CodeGroup>
```bash cURL
curl -G "http://localhost:8000/products/search" \
  -H "Authorization: Bearer $TOKEN" \
  --data-urlencode "q=cozy green jacket" \
  --data-urlencode "limit=20"
```

```python Python
import requests

resp = requests.get(
    "http://localhost:8000/products/search",
    headers={"Authorization": f"Bearer {TOKEN}"},
    params={"q": "cozy green jacket", "limit": 20},
)
resp.raise_for_status()
results = resp.json()
```

```javascript JavaScript
const params = new URLSearchParams({ q: "cozy green jacket", limit: "20" });
const res = await fetch(`http://localhost:8000/products/search?${params}`, {
  headers: { Authorization: `Bearer ${token}` },
});
const results = await res.json();
```
</CodeGroup>

## Response — 200 OK

An array of products, most relevant first. Each item is a full product plus a
`relevance` score.

```json
[
  {
    "id": 42,
    "status": "ready",
    "content_type": "clothing",
    "name": "forest-green wool jacket",
    "description": "A cozy oversized forest-green wool jacket on a store rack.",
    "tags": "cozy, oversized, forest green, cream, in-store rack",
    "image_url": "data:image/jpeg;base64,/9j/...",
    "relevance": 0.82,
    "board_ids": [1],
    "created_at": "2026-07-04T02:39:26.359572-07:00"
  }
]
```

<ParamField body="relevance" type="float">
  Cosine similarity between the query and the product embedding
  (`1 - cosine_distance`), roughly in `[0, 1]`. Higher is more relevant.
  Purely informative — results are already sorted, with no server-side
  threshold, so a live gallery can reorder as the user types without ever
  blanking out.
</ParamField>

All other fields match [`GET /products/{id}`](/api-reference/products#fields).

## How indexing works

Every `ready` capture is embedded at capture time (best-effort) into a single
1024-dim multimodal vector combining its composed text and cropped image —
see the [Embedding Service](/services/embedding-service). The query side uses
the **same** model with `input_type="query"`, so query and product vectors
live in the same space.

Query embeddings are cached (LRU) so search-as-you-type does not re-embed the
same text on every keystroke.

## Error responses

### 401 Unauthorized

Missing or invalid bearer token.

### 422 Unprocessable Entity

`q` is empty/whitespace, or `limit`/`offset` are out of bounds.

### 503 Service Unavailable

Semantic search is not configured (no `VOYAGE_API_KEY`). Capture still works;
products are just left unindexed until a key is present and the backfill runs.

```json
{ "detail": "Semantic search is currently unavailable" }
```

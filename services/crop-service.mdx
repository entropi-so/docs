## Overview

The **Crop Service** takes the bounding box returned by the
[OCR + Bbox Service](/services/ocr-service) and crops the screenshot
locally using Pillow. There is **no AI call here** — bbox detection
happens upstream in the unified Gemini call.

**File**: `app/services/vision/image_crop_service.py`

## Function

<ParamField body="crop_with_bbox" type="async function">
  Crops the image to the product bbox returned by
  `extract_and_locate_product`. Returns the cropped image as base64
  JPEG.
</ParamField>

```python
async def crop_with_bbox(
    base64_image: str,
    bbox_normalized: list[int],
    padding_percent: float = 0.20,
) -> str
```

<ParamField body="base64_image" type="string" required>
  Base64-encoded full screenshot.
</ParamField>
<ParamField body="bbox_normalized" type="[ymin, xmin, ymax, xmax]" required>
  Normalized bbox (0–1000) from the OCR + Bbox Service.
</ParamField>
<ParamField body="padding_percent" type="float" optional>
  Padding around the bbox, as a fraction of bbox size (default 0.20).
</ParamField>

**Returns**: Base64-encoded cropped JPEG. **On any failure, returns the
original `base64_image` unchanged** (graceful degradation).

## How It Works

<Steps>
  <Step title="1. Decode and inspect">
    Pillow opens the image and reads its dimensions.
  </Step>

  <Step title="2. Convert normalized bbox to pixels">
    `[ymin, xmin, ymax, xmax]` (0–1000) is mapped to pixel coordinates using image width/height.
  </Step>

  <Step title="3. Apply padding and clamp">
    Padding is added around the bbox; coordinates are clamped to the image bounds.
  </Step>

  <Step title="4. Crop and re-encode">
    Pillow crops the region and re-encodes to JPEG (quality 95), returned as base64.
  </Step>

  <Step title="5. Graceful fallback">
    If anything raises — invalid bbox, decode error, crop error — the original `base64_image` is returned unchanged.
  </Step>
</Steps>

## Concurrency

Pillow operations are CPU-bound and synchronous. The service wraps the
crop in `asyncio.to_thread` so concurrent pipeline runs do not block the
event loop.

## Usage Example

```python
from app.services.vision.image_crop_service import crop_with_bbox

cropped_image = await crop_with_bbox(
    base64_image=base64_image,
```

## Error Handling

<Warning>
  The service implements **graceful degradation**:
  - Invalid bbox (`null`, out of range, zero area) → returns original
  - Decode/crop exception → returns original
  - The pipeline always continues to enrichment, even when the crop
    falls back to the original image.
</Warning>

## Coordinate System

The bbox uses Gemini's normalized 0–1000 system:

- `[0, 0, 0, 0]` = empty (degenerate; falls back to original)
- `[ymin, xmin, ymax, xmax]` = top-left → bottom-right, with `y` first

The service converts these to pixel coordinates using actual image
dimensions, then applies `padding_percent` and clamps to image bounds.

## Overview

The **OCR + Bbox Service** is a single Gemini 2.5 Flash call that
extracts structured product information **and** the product's bounding
box in one shot. It replaces the previous two-call flow (separate OCR
then bbox detection), saving roughly 3 seconds per pipeline run.

**File**: `app/services/vision/ocr_service.py`

## Function

<ParamField body="extract_and_locate_product" type="async function">
  Analyzes a base64-encoded image and returns product fields plus a
  normalized bounding box in a single Gemini call.
</ParamField>

```python
async def extract_and_locate_product(base64_image: str) -> Optional[dict]
```

<ParamField body="base64_image" type="string" required>
  Base64-encoded JPEG image (from Image Service)
</ParamField>

**Returns**: Dictionary with product fields and `bbox`, or `None` if extraction fails.

## Output Schema

```json
{
  "name": "Headliner (Gen 2)",
  "price": 459.0,
  "currency": "EUR",
  "description": "Smart sunglasses with built-in camera and audio.",
  "merchant": "Ray-Ban | Meta",
  "category": "eyewear",
  "tags": "sunglasses, smart, audio",
  "object_type": "sunglasses",
  "url": null,
  "bbox": [120, 85, 870, 940]
}
```

<ParamField body="bbox" type="[ymin, xmin, ymax, xmax] | null">
  Normalized bounding box of the product in the image. Coordinates are
  integers in the `0–1000` range (Gemini's native bbox format). `null` if
  the model could not locate the product — in that case the downstream
  crop step returns the original image unchanged.
</ParamField>

The remaining fields (`name`, `price`, `currency`, `description`,
`merchant`, `category`, `tags`, `object_type`, `url`) match the
previous OCR-only schema.

## Concurrency

The Gemini Python SDK call is synchronous. The service wraps it in
`asyncio.to_thread`, so concurrent `/pipeline/analyze` requests do not
serialize on the FastAPI event loop.

## Usage Example

```python
from app.services.vision.image_service import prepare_image_for_ocr
from app.services.vision.ocr_service import extract_and_locate_product

async def process_screenshot(file: UploadFile):
    image_bytes = await file.read()
    base64_image = await prepare_image_for_ocr(image_bytes, file.content_type)

    ocr_data = await extract_and_locate_product(base64_image)
    if not ocr_data or not ocr_data.get("name"):
        raise ValueError("OCR failed to extract product name")

    return ocr_data  # includes "bbox" for the local crop step
```

## Error Handling

<Warning>
  The service returns `None` if the API call fails or the response is
  unparseable. In the pipeline this puts the product into `stumped`
  status with an `error_message`.

  If the call succeeds but `bbox` is `null`, the crop step gracefully
  returns the original image and the pipeline continues to enrichment.
</Warning>

## System Prompt

The service uses `PRODUCT_EXTRACT_AND_LOCATE_PROMPT`, defined in
`app/core/prompts.py`. The prompt instructs Gemini to:

- Extract all product fields in one structured JSON
- Return a single bbox `[ymin, xmin, ymax, xmax]` (0–1000) for the
  primary product, or `null` if no product is clearly identifiable
- Avoid wrapping the bbox in any additional metadata

## Limitations

<AccordionGroup>
  <Accordion title="Image Quality">
    Works best with clear, high-resolution screenshots. Blurry or low-quality images may reduce accuracy.
  </Accordion>

  <Accordion title="Multiple Products">
    Returns a single bbox for the dominant product. Cluttered screenshots may yield `bbox = null`.
  </Accordion>

  <Accordion title="Language Support">
    Primarily optimized for English and French.
  </Accordion>
</AccordionGroup>

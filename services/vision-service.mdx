## Overview

The **Vision Service** is the heart of the pipeline: a single Gemini 2.5
Flash multimodal call that performs the whole local understanding — object
recognition, text reading, a structured lifestyle record, and the subject's
bounding box — in one shot. No OCR engine or object detector on the side, and
no web.

**File**: `app/services/vision/vision_service.py`

## Function

<ParamField body="analyze_capture" type="async function">
  Analyzes a base64-encoded image and returns the lifestyle record plus a
  normalized bounding box in a single Gemini call.
</ParamField>

```python
async def analyze_capture(base64_image: str) -> Optional[dict]
```

<ParamField body="base64_image" type="string" required>
  Base64-encoded JPEG image (from the Image Service).
</ParamField>

**Returns**: A dictionary with the record fields and `bbox`, or `None` if the
call fails (which puts the product into `stumped`, keeping the raw photo).

## Output Schema

```json
{
  "content_type": "clothing",
  "name": "forest-green wool jacket",
  "description": "A cozy oversized forest-green wool jacket on a store rack, soft natural light.",
  "category": "outerwear",
  "object_type": "wool jacket",
  "colors": ["forest green", "cream"],
  "style_tags": ["cozy", "oversized"],
  "setting": "in-store rack",
  "brand": "Sézane",
  "brand_confidence": 0.9,
  "raw_text": "SÉZANE",
  "price": 145.0,
  "currency": "EUR",
  "bbox": [100, 100, 900, 900]
}
```

<ParamField body="content_type" type="string">
  Coarse lifestyle bucket: `place`, `clothing`, `decor`, `food`, or `other`.
</ParamField>

<ParamField body="brand / brand_confidence" type="string | null / float | null">
  The brand is returned **only when it is actually readable** in the image (a
  logo, a label, visible text), with a confidence. It is never guessed from
  style. Downstream, the brand is kept in `merchant` only when
  `brand_confidence` clears the threshold (0.5); otherwise both are dropped.
</ParamField>

<ParamField body="colors / style_tags / setting" type="list / list / string">
  Folded into the product's `tags` column downstream (comma-separated,
  deduped), so they are carried and contribute to the search embedding.
</ParamField>

<ParamField body="bbox" type="[ymin, xmin, ymax, xmax] | null">
  Normalized bounding box of the main subject. Integers in the `0–1000` range
  (Gemini's native format). `null` if no subject is locatable — the crop step
  then returns the original image unchanged.
</ParamField>

## Describing the unknown

The prompt explicitly instructs the model to describe items it cannot
identify. An unfamiliar jacket in a store still yields a rich record
(`content_type`, `description`, `colors`, `object_type`), which is what lets
it live in a moodboard and resurface under a query like *"cozy green jacket"*.
It returns `null` for fields it genuinely cannot determine rather than
hallucinating.

## Concurrency

The Gemini Python SDK call is synchronous. The service wraps it in
`asyncio.to_thread`, so concurrent `/pipeline/analyze` requests do not
serialize on the FastAPI event loop.

## Usage Example

```python
from app.services.vision import analyze_capture, prepare_image_for_ocr

async def understand(file):
    image_bytes = await file.read()
    base64_image = await prepare_image_for_ocr(image_bytes, file.content_type)

    record = await analyze_capture(base64_image)
    if not record:
        raise ValueError("Visual understanding returned no data")

    bbox = record.pop("bbox", None)  # consumed by the crop step
    return record, bbox
```

## Error Handling

<Warning>
  The service returns `None` if the API call fails or the response is
  unparseable. In the pipeline this persists the product as `stumped`,
  keeping the raw photo in `image_url` and recording an `error_message`.

  If the call succeeds but `bbox` is `null`, the crop step returns the
  original image and the pipeline continues to `ready`.
</Warning>

## System Prompt

The service uses `VISUAL_UNDERSTANDING_PROMPT`, defined in
`app/core/prompts.py`. The prompt instructs Gemini to:

- Produce a rich, search-native record for lifestyle content (places, clothing, decor, food)
- Describe even unidentified items richly, returning `null` only when a field is genuinely unknown
- Return the brand only when it is readable, with a confidence
- Emit a single `bbox [ymin, xmin, ymax, xmax]` (0–1000) for the main subject, or `null`

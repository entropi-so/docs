## The problem

People discover things everywhere — a jacket in a store, a café on Instagram, a lamp in a friend's photo, a dish on a menu. By the time they want to find it again, the source is buried, the name was never known, or they cannot remember where they saw it.

The meaning is in the image. Describing it by hand is tedious. Finding it again later is unreliable.

## What Entropi does

Entropi is an HTTP API for lifestyle capture. You upload a screenshot or a camera photo of a place, an item of clothing, a piece of decor, or a dish, and Entropi returns a rich, structured record of what it sees: a natural description, a content type, colors and style, the brand when it is actually readable, and a clean cropped image.

The capture is **synchronous**: a single `POST /pipeline/analyze` runs the whole understanding locally and returns the finished product in the response — no polling, no background jobs. Every record is also indexed for **semantic search**, so a later query like *"cozy green jacket"* finds the item even if those exact words appear nowhere in its data.

## Mission

Save anything you see. Find it instantly. Come back to it when you want.

We exist to remove the friction between visual discovery and memory. Every place, outfit, or object anyone has ever shown you should be one search away.

## Who this is for

This documentation is written for the people who build, run, and integrate against Entropi:

- **Engineers** — backend, mobile, and frontend developers integrating the API or contributing to it.
- **Operators** — anyone deploying or self-hosting Entropi.
- **Partners** — external teams calling the public API.

It is not a marketing site. It assumes you know how to read a `curl` command, run a Python script, and interpret an HTTP status code.

## What you can do in five minutes

By the end of the [Quickstart](/getting-started/quickstart) you will have:

1. Registered a user and obtained a JWT.
2. Created a list.
3. Captured a product from an image — and received the finished record in the response.
4. Searched your saved products by free text.

If something does not work in those five minutes, that is a documentation bug. Open an issue.

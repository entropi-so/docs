## Software

| Component       | Minimum version | Notes                                                              |
| --------------- | --------------- | ------------------------------------------------------------------ |
| Python          | 3.10            | 3.11 and 3.12 also supported.                                      |
| PostgreSQL      | 14              | Any 14+ release is fine.                                           |
| `libheif`       | latest          | Required by `pillow-heif` for HEIC decoding. Install via OS package manager. |
| `git`           | any             | For cloning the repository.                                        |

## API credentials

Entropi calls Gemini and Groq directly. Provision keys before starting the install:

- **Gemini API** — [Google AI Studio](https://aistudio.google.com/app/apikey). Used for OCR + bbox extraction and grounded search enrichment.
- **Groq API** — [Groq Console](https://console.groq.com). Used by the advisor and URL import services.
- **Google OAuth** — Google Cloud Console. Required for sign-in.

You can reuse a single Gemini key across `GEMINI_VISION_API_KEY` and `GEMINI_SEARCH_API_KEY`.

## Hardware

Entropi is I/O-bound (the heavy work happens at provider APIs), so hardware requirements are modest. For a single-node deployment:

- **CPU**: 2 cores minimum, 4 recommended.
- **Memory**: 1&nbsp;GB minimum, 2&nbsp;GB recommended.
- **Disk**: 1&nbsp;GB for the application; PostgreSQL grows with product volume.

## Network

The host must reach:

- `generativelanguage.googleapis.com` (Gemini API)
- `api.groq.com` (Groq API)
- `accounts.google.com` and `oauth2.googleapis.com` (Google OAuth)
- The PostgreSQL instance (local or managed)

Outbound HTTPS on port 443 must be permitted to the AI providers.

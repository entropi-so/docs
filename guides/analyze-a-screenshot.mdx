title: Capture a screenshot or photo
description: "Submit an image to the pipeline and get back a finished, structured product."
---

## Context

You have a screenshot or a camera photo of a place, an outfit, a piece of decor, or a dish, and you want Entropi to turn it into a structured record. This is the canonical use case.

The flow is a **single** HTTP call: a `POST` that returns the finished product. No polling.

## Steps

<Steps>
  <Step title="Authenticate">
    You need a JWT. See [Authentication](/api-reference/authentication) if you do not have one yet.

    ```bash
    TOKEN=$(curl -s -X POST http://localhost:8000/auth/login \
      -H "Content-Type: application/json" \
      -d '{"username":"you@example.com","password":"..."}' \
      | jq -r .access_token)
    ```
  </Step>

  <Step title="Pick a list">
    Every product belongs to a list. If you do not have one yet, create it.

    ```bash
    LIST_ID=$(curl -s -X POST http://localhost:8000/lists/ \
      -H "Authorization: Bearer $TOKEN" \
      -H "Content-Type: application/json" \
      -d '{"name":"Inbox"}' \
      | jq -r .id)
    ```
  </Step>

  <Step title="Capture the image">
    `POST /pipeline/analyze` accepts multipart/form-data with `list_id` and `file` fields, and returns the finished product.

    ```bash
    curl -s -X POST http://localhost:8000/pipeline/analyze \
      -H "Authorization: Bearer $TOKEN" \
      -F "list_id=$LIST_ID" \
      -F "file=@capture.jpg" | jq
    ```

    Accepted formats: HEIC, JPEG, PNG, WebP. Maximum size: 10&nbsp;MB. Set your
    client timeout above the server cap (45&nbsp;s) and show a loading state
    while it runs.
  </Step>
</Steps>

## Expected result

A `ready` product looks like this:

```json
{
  "id": 42,
  "status": "ready",
  "content_type": "clothing",
  "name": "forest-green wool jacket",
  "description": "A cozy oversized forest-green wool jacket on a store rack.",
  "category": "outerwear",
  "object_type": "wool jacket",
  "tags": "cozy, oversized, forest green, cream, in-store rack",
  "merchant": "Sézane",
  "brand_confidence": 0.9,
  "raw_text": "SÉZANE",
  "price": 145.0,
  "currency": "EUR",
  "product_url": null,
  "image_url": "data:image/jpeg;base64,/9j/...",
  "error_message": null,
  "board_ids": [42],
  "created_at": "2026-07-04T02:39:26.359572-07:00"

```

Once it is saved, it is also indexed for [semantic search](/guides/semantic-search).

## Common mistakes

- **Forgetting `list_id`** — the request returns 422 with no product created.
- **Submitting an image larger than 10&nbsp;MB** — returns 413 with no product created.
- **Expecting a `product_id` to poll** — there is no polling; the response *is* the finished product.
- **Treating `stumped` as a lost capture** — a `stumped` product still exists, with the raw photo in `image_url`. Surface it as "we could not read this one" and let the user retry or keep it.
- **Expecting a `product_url`** — capture does no web lookup; `product_url` is always `null`.

title: Quickstart
description: "Register, log in, capture a product, and search your library. Five minutes, no assumptions."
---

This page assumes Entropi is running locally on `http://localhost:8000`. If it is not, follow the [Self-Hosting installation guide](/self-hosting/installation) first.

You will need `curl` and `jq`.

<Steps>
  <Step title="Register a user">
    ```bash
    curl -s -X POST http://localhost:8000/auth/register \
      -H "Content-Type: application/json" \
      -d '{
        "username": "you",
        "email": "you@example.com",
        "password": "a-strong-password"
      }'
    ```

    A `201 Created` response confirms the user exists.
  </Step>

  <Step title="Log in and store the token">
    ```bash
    TOKEN=$(curl -s -X POST http://localhost:8000/auth/login \
      -H "Content-Type: application/json" \
      -d '{"username":"you@example.com","password":"a-strong-password"}' \
      | jq -r .access_token)
    ```

    The token is a JWT. It expires after 7 days by default.
  </Step>

  <Step title="Create a list">
    Every product belongs to a list (a board, a wishlist, a project — whatever the client calls it).

    ```bash
    LIST_ID=$(curl -s -X POST http://localhost:8000/lists/ \
      -H "Authorization: Bearer $TOKEN" \
      -H "Content-Type: application/json" \
      -d '{"name":"Quickstart"}' \
      | jq -r .id)
    ```
  </Step>

  <Step title="Capture a product">
    ```bash
    curl -s -X POST http://localhost:8000/pipeline/analyze \
      -H "Authorization: Bearer $TOKEN" \
      -F "list_id=$LIST_ID" \
      -F "file=@capture.jpg" | jq
    ```

    This is **synchronous**: the call runs the whole pipeline and returns the
    finished product (typically within a few seconds, capped at 45&nbsp;s).
    The response `status` is `ready` — or `stumped` if understanding failed,
    in which case the raw photo is still saved. No polling.
  </Step>

  <Step title="Search your library">
    ```bash
    curl -s -G "http://localhost:8000/products/search" \
      -H "Authorization: Bearer $TOKEN" \
      --data-urlencode "q=cozy green jacket" | jq
    ```

    Free-text semantic search over your own products, ranked by relevance.
    It matches on what the photo shows, not just stored words.
  </Step>
</Steps>

## What you just did

You exercised the full Entropi flow: authentication, list creation, synchronous capture, and semantic search.

`POST /pipeline/analyze` runs image prep, visual understanding (one Gemini call), crop, and embedding **in the request**, then returns the finished product. There is nothing to poll — the record is complete when you get the response.
